🎯 Partie 3 — Exemple complet : version POO de generateur.py

Réécriture du module generateur.py en utilisant la Programmation Orientée Objet.

1. Classe De

import random

class De:
    """Représente un dé à 6 faces."""

    NB_FACES = 6

    def __init__(self):
        self.valeur = 1

    def lancer(self):
        """Lance le dé et retourne la valeur obtenue (1-6)."""
        self.valeur = random.randint(1, self.NB_FACES)
        return self.valeur

2. Classe LancerDouble

class LancerDouble:
    """Gère le lancer de deux dés et teste s'ils sont égaux."""

    def __init__(self):
        self.de1 = De()
        self.de2 = De()
        self.egaux = False
        self.somme = 0

    def effectuer(self):
        """Lance les deux dés, retourne les valeurs et l'égalité."""
        v1 = self.de1.lancer()
        v2 = self.de2.lancer()
        self.egaux = (v1 == v2)
        self.somme = v1 + v2
        return v1, v2, self.egaux, self.somme

3. Classe Simulation

class Simulation:
    """Lance N lancers de deux dés et calcule la fréquence de doubles."""

    def __init__(self, nb_experiences):
        self.nb_experiences = nb_experiences
        self.nb_doubles = 0
        self.frequence = 0.0

    def executer(self):
        """Exécute la simulation complète."""
        self.nb_doubles = 0
        for _ in range(self.nb_experiences):
            lancer = LancerDouble()
            _, _, egaux, _ = lancer.effectuer()
            if egaux:
                self.nb_doubles += 1
        self.frequence = self.nb_doubles / self.nb_experiences
        return self.frequence

4. Utilisation

# Exemple d'utilisation
simu = Simulation(10000)
freq = simu.executer()
print(f"Fréquence des doubles : {freq:.4f}")
print(f"Probabilité théorique    : {1/6:.4f}")

5. Avantages de la version POO

  • Réutilisable : on peut créer plusieurs dés, simulations, etc.
  • Extensible : facile d'ajouter un DePondere par héritage.
  • Lisible : le code est organisé en blocs logiques.
  • Testable : chaque classe peut être testée indépendamment.
  • Maintenable : les modifications sont localisées dans une classe.

💡 Comparaison : la version procédurale de generateur.py faisait tout avec des fonctions isolées. La version POO regroupe données et comportements dans des classes, ce qui facilite la réutilisation et l'extension.

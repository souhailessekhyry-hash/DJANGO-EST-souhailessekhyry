🧩 Partie 2 — Programmation Orientée Objet

La POO est un paradigme qui permet d'organiser le code en regroupant données et comportements dans des classes.

1. Classe et objet

Une classe est un modèle qui définit la structure et le comportement d'un objet. Un objet est une instance d'une classe. 

class De:
    """Modèle d'un dé à 6 faces."""

    def __init__(self, faces=6):
        self.faces = faces      # Attribut d'instance
        self.valeur = 1

    def lancer(self):
        """Méthode : simule le lancer du dé."""
        import random
        self.valeur = random.randint(1, self.faces)
        return self.valeur

# Création d'objets (instances)
de1 = De()      # dé à 6 faces
de2 = De(20)    # dé à 20 faces

# Appel de méthode
resultat = de1.lancer()
print(f"Le dé 1 donne {resultat}")

2. Attributs et méthodes

Les attributs stockent l'état de l'objet. Les méthodes définissent son comportement. 

class LancerDouble:
    """Gère le lancer de deux dés et l'analyse du résultat."""

    def __init__(self, de1, de2):
        self.de1 = de1
        self.de2 = de2
        self.somme = 0
        self.double = False

    def effectuer(self):
        valeur1 = self.de1.lancer()
        valeur2 = self.de2.lancer()
        self.somme = valeur1 + valeur2
        self.double = (valeur1 == valeur2)
        return valeur1, valeur2

    def est_double(self):
        return self.double

    def afficher(self):
        print(f"{self.de1.valeur} + {self.de2.valeur} = {self.somme}")
        if self.double:
            print("→ Double !")

3. Héritage

L'héritage permet à une classe de réutiliser le code d'une autre classe. 

class DePondere(De):
    """Dé truqué qui hérite de De."""

    def __init__(self, faces=6, poids=None):
        super().__init__(faces)     # Appel du constructeur parent
        self.poids = poids or [1] * faces

    def lancer(self):
        """Surcharge de la méthode lancer avec pondération."""
        import random
        self.valeur = random.choices(
            range(1, self.faces + 1),
            weights=self.poids,
            k=1
        )[0]
        return self.valeur

4. POO en Django

Django utilise intensivement la POO pour ses Models, ses Vues basées sur les classes, et ses Forms. 

# Modèle Django (models.py) — héritage de django.db.models.Model
from django.db import models

class SimulationResult(models.Model):
    """Table en base de données via héritage de Model."""
    nb_experiences = models.IntegerField()
    frequence = models.FloatField()
    date = models.DateTimeField(auto_now_add=True)

    def __str__(self):
        return f"Simulation n°{self.id} — {self.frequence}"

# Vue basée sur une classe (cbv)
from django.views.generic import ListView

class SimulationListView(ListView):
    model = SimulationResult
    template_name = "simulations/liste.html"
    context_object_name = "simulations"
    ordering = ["-date"]

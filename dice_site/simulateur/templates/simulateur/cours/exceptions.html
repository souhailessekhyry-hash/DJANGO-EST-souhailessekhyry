⚠️ Partie 1 — Gestion des exceptions

Les exceptions sont des erreurs qui surviennent pendant l'exécution d'un programme. Python permet de les capturer et de les traiter pour éviter que le programme ne plante.

1. Le bloc try / except

Le bloc try contient le code susceptible de générer une erreur. Le bloc except permet de gérer l'erreur si elle se produit. 

try:
    # Code potentiellement dangereux
    nombre = int(input("Entrez un nombre : "))
    resultat = 10 / nombre
    print(f"Résultat : {resultat}")
except ZeroDivisionError:
    # Gestion de l'erreur de division par zéro
    print("Erreur : Division par zéro interdite !")
except ValueError:
    # Gestion de l'erreur de type incorrect
    print("Erreur : Veuillez entrer un nombre valide.")

2. L'exception ValueError

ValueError est levée quand une fonction reçoit un argument de type correct mais de valeur inappropriée. 

# Exemple classique de ValueError
try:
    age = int(input("Quel âge avez-vous ? "))
    if age < 0:
        raise ValueError("L'âge ne peut pas être négatif.")
except ValueError as e:
    print(f"Erreur : {e}")

3. Gestion des erreurs en Django

Django offre plusieurs mécanismes pour gérer les erreurs dans les vues et les templates : 

# Dans une vue Django (views.py)
from django.shortcuts import render, get_object_or_404
from django.http import Http404

def detail_simulation(request, simulation_id):
    try:
        simulation = SimulationResult.objects.get(id=simulation_id)
    except SimulationResult.DoesNotExist:
        raise Http404("Cette simulation n'existe pas.")
    except Exception as e:
        return render(request, "erreur.html", {"message": str(e)})
    return render(request, "detail.html", {"simulation": simulation})

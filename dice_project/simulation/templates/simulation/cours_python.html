⚠️ A. Gestion des exceptions

Les exceptions sont des erreurs qui surviennent pendant l'exécution. Python permet de les capturer pour éviter que le programme ne plante.

1. try / except

try contient le code dangereux, except gère l'erreur.

try:
    nombre = int(input("Entrez un nombre : "))
    resultat = 10 / nombre
    print(f"Résultat : {resultat}")
except ZeroDivisionError:
    print("Division par zéro interdite !")
except ValueError:
    print("Veuillez entrer un nombre valide.")

2. ValueError

Levee quand une fonction reçoit une valeur inappropriée.

try:
    age = int(input("Âge : "))
    if age < 0:
        raise ValueError("Âge négatif impossible.")
except ValueError as e:
    print(f"Erreur : {e}")

3. Gestion des erreurs Django

# Vue Django avec gestion d'erreur
from django.http import Http404

def detail_simulation(request, id):
    try:
        sim = SimulationResult.objects.get(id=id)
    except SimulationResult.DoesNotExist:
        raise Http404("Simulation introuvable.")
    except Exception as e:
        return render(request, "erreur.html", {"msg": str(e)})
    return render(request, "detail.html", {"sim": sim})

🧱 B. Programmation Orientée Objet

La POO organise le code autour d'objets qui ont des attributs (données) et des méthodes (comportements).

1. Classes et objets

Une classe est un modèle. Un objet est une instance.

class De:
    """Représente un dé à 6 faces."""
    def __init__(self, faces=6):
        self.faces = faces
        self.valeur = None

    def lancer(self):
        self.valeur = random.randint(1, self.faces)
        return self.valeur

# Création d'objets
de1 = De()
de2 = De(6)
print(de1.lancer())  # 4 par exemple

2. Attributs et méthodes

Attributs = variables de l'objet. Méthodes = fonctions de l'objet.

class Simulation:
    proba_theorique = 1 / 6  # Attribut de classe

    def __init__(self, nb):
        self.nb = nb           # Attribut d'instance
        self.frequence = 0.0

    def executer(self):        # Méthode
        succes = 0
        for _ in range(self.nb):
            d1, d2 = random.randint(1,6), random.randint(1,6)
            if d1 == d2: succes += 1
        self.frequence = succes / self.nb
        return self.frequence

3. Héritage simple

Une classe fille hérite de la classe mère.

class De:
    def lancer(self):
        return random.randint(1, 6)

class DePondere(De):  # Héritage
    def __init__(self, valeur_forcee=None):
        self.valeur_forcee = valeur_forcee

    def lancer(self):  # Surcharge
        if self.valeur_forcee:
            return self.valeur_forcee
        return super().lancer()

4. POO dans Django Models

Chaque modèle Django est une classe qui hérite de models.Model.

from django.db import models

class SimulationResult(models.Model):
    # Attributs = champs de la table
    nb_experiences = models.IntegerField()
    frequence = models.FloatField()
    date = models.DateTimeField(auto_now_add=True)

    def __str__(self):
        return f"{self.nb_experiences} → {self.frequence:.4f}"

    def est_proche(self):
        return abs(self.frequence - 1/6) < 0.01
